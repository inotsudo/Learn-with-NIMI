## How To Use This Document

Every item must be marked PASS, FAIL, or N/A before launch. Any FAIL in a Critical section blocks launch. Any FAIL in a High section requires owner sign-off to proceed.

```
✅ PASS — Verified and working
❌ FAIL — Broken or missing (blocks launch if Critical)
⚠️ PARTIAL — Works but incomplete (owner decision)
➖ N/A — Not applicable for this launch
```

## ROLLBACK PLAN

If critical issues discovered post-launch:

| Severity | Action | Time |
|---|---|---|
| UI bug | Hotfix deploy via Vercel | 15-30 min |
| Data corruption | PITR restore + redeploy | 1-2 hours |
| Security vulnerability | Take down + fix + redeploy | 1-4 hours |
| Content issue | Unpublish story via admin | 1 minute |
| Full rollback | Revert Vercel to previous deployment | 2 minutes |

Vercel maintains deployment history. Any previous deployment can be promoted to production instantly.
